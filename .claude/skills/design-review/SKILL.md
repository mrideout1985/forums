# Design Review Approach

When reviewing or fixing UI/design issues:

1. **Gather context**: framework, styling method, review scope
2. **Inspect visually**: look for overflow, overlap, alignment, spacing, responsive breakage, contrast, focus states
3. **Test viewports**: 375px (mobile), 768px (tablet), 1280px (desktop), 1920px (wide)
4. **Prioritize**: P1 (breaks functionality) → P2 (degrades UX) → P3 (minor inconsistencies)
5. **Minimal changes**: respect existing patterns, fix one issue at a time, verify before moving on
6. **Confirm before large-scale changes**
